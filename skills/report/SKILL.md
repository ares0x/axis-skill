# Report — 输出报告协议

当 `/analyze` 推理分析阶段完成，调用此 subskill 生成最终的 Markdown 报告。

## 核心质量与法务质量闸门（硬性要求）

1. **置顶免责声明（强制）**：
   在报告首屏大标题正下方，必须紧跟一段引用块形式的免责声明：
   > 🚨 **【AI 辅助说明与免责声明】**
   > 本报告内容均由 AI 辅助生成，仅供参考，不构成任何填报、录取或决策的承诺与建议。最终决定请你和家人结合官方信息综合判断。
   > 录取位次、招生计划等核心数据具有时效性与大小年波动风险，具体填报事项请前往 **[{考试院全称}]({官网URL})** 查看和操作。
   *(注：官方机构名称与官网 URL 必须根据考生的省份动态匹配替换)*

2. **数据来源溯源表格（强制）**：
   报告末尾必须另起一段，以列表/表格形式对所有硬性数据（控制线、一分一段、战略撤销等）列出具体的源站超链接。格式必须为：
   `- {数据项}：[{页面名}]({URL}) · 采集年份：{YYYY} · 采集于 {YYYY-MM-DD}`
   - 严禁使用聚合站的二次链接或首页 URL 占位，必须是具体页面级 URL。

3. **口径清洗（黑名单词汇）**：
   报告中绝对禁止出现以下承诺性词汇：
   - “保证录取”、“一定能录” -> 统一改为“录取概率较大”
   - “100%录取” -> 统一改为“录取概率极高”
   - “法律效力”、“担保” -> 统一改为“效力参考”
