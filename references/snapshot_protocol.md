# Snapshot — 存档与恢复协议

本文件合并了 save、restore、list 三个操作，统一管理考生评估进度的存档与恢复流程。

## Save — 评估进度存档

### 触发形式

| 命令/输入 | 行为 |
|---|---|
| `/save` | 存当前对话的诊断状态。标题自动从对话上下文提取 |
| `/save <title>` | 指定标题，比如 `/save 选科偏科查杀` |
| 「保存当前状态」「记下来」「存档」 | 等价于 `/save` |

### 工作流程

1. **检查当前活跃考生**：
   - 确认当前对话是否存在已初始化的考生 `uid`。若没有，请提醒用户先进行 `/init [uid]`。
2. **提取标题**：
   - 从对话中提取一句能够代表当前决策演进的短句（20字以内）。
   - 例如：「想学软件但怕替代」、「放弃传统会计」、「艺术设计对冲具身智能」。
3. **执行命令**：
   - 必须将 `/init` 和 `/save` 链式传入同一进程（runner.py 是无状态 CLI，单独调用 `/save` 会因 current_uid 为空而失败）：
   ```bash
   python3 scripts/runner.py "/init {uid}" "/save {title}"
   ```
4. **给用户简短确认**：
   - 仅输出一行确认，指示保存的文件位置。

## Restore — 接续评估

### 触发形式

| 命令/输入 | 行为 |
|---|---|
| `/restore` | 恢复当前项目下最新的一份快照 |
| `/restore <序号>` | 恢复指定序号的快照（由 `/list` 呈现的编号） |
| `/restore <模糊名称>` | 模糊匹配快照文件名或标题进行恢复 |
| 「接着上次」「之前评估的结论」「续上」 | 等价于 `/restore` |

### 工作流程

1. **检查与定位**：
   - 运行 `python3 scripts/runner.py "/init {uid}" "/list"` 检查当前考生的快照列表，或使用用户的指定参数。
2. **执行恢复**：
   - 运行命令行：`python3 scripts/runner.py "/init {uid}" "/restore {arg}"` 恢复 facts.json 的数据与 Target Majors。
   - 必须将 `/init` 和 `/restore` 链式传入同一进程（runner.py 是无状态 CLI）。
3. **呈现状态**：
   - 展现恢复后的当前画像数据（分数、选科组合、霍兰德代码、目标专业池），并向用户询问从哪里继续探讨。

## List — 列出历史存档

### 触发形式

| 命令/输入 | 行为 |
|---|---|
| `/list` | 列出所有快照和考生会话 |
| 「有哪些学生」「看看我的存档」 | 等价于 `/list` |

### 工作流程

1. **执行查询**：
   - 运行命令行：`python3 scripts/runner.py "/init {uid}" "/list"` （需链式传入 /init 以激活考生会话）
   - 若无活跃考生，可直接运行 `python3 scripts/runner.py /list` 仅列出所有会话目录。
2. **格式化输出**：
   - 向用户清晰列出所有已存在的快照序号、保存时间、标题与完成阶段，以及其他可切换的考生 UID 目录。
